# Contributing to EDNA OS

Thank you for your interest in EDNA OS.

This repository documents an engineering project rather than a traditional software product.

Consistency is more important than speed.

# Engineering Principles

Every contribution should reinforce:

- Engineering before software
- Measurement before optimization
- Operational evidence
- Deterministic control
- Commercial orchard reality

Avoid marketing language.

Avoid unnecessary complexity.

---

# Code Guidelines

- Use TypeScript.
- Prefer Server Components.
- Use semantic HTML.
- Follow the Design System.
- Build reusable components.
- Keep business logic outside UI components.
- Avoid duplication.
- Avoid `any`.

---

# Content Guidelines

Write as an engineer.

Do not write as a marketer.

Prefer:

- measured
- observed
- validated
- engineered
- operational

Avoid:

- revolutionary
- AI-powered
- next-generation
- industry-leading
- game-changing

---

# Images

Use only filenames defined in

```
CODEX/05_IMAGE_CONVENTION.md
```

Production images should be original engineering photography whenever possible.

---

# Pull Requests

Keep pull requests focused.

One logical change per pull request.

Large architectural changes should first be reflected in the documentation inside `CODEX/`.

---

# Documentation

If implementation changes the engineering architecture,

update the corresponding document inside

```
CODEX/
```

Documentation and implementation should remain synchronized.

---

# Goal

The objective is to build a long-term engineering knowledge base documenting the creation of the Orchard Operating System.

Every contribution should strengthen that objective.

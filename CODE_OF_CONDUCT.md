# Code of Conduct

## Our Goal

EDNA OS is an engineering project documenting the development of the Orchard Operating System.

The purpose of this repository is to encourage constructive technical discussion, knowledge sharing and collaborative engineering.

We welcome contributions that improve the quality, accuracy and maintainability of the project.

---

## Expected Behavior

Contributors are expected to:

- Treat others with professionalism and respect.
- Focus discussions on engineering and technical topics.
- Provide constructive feedback.
- Support technical arguments with evidence whenever possible.
- Respect differing opinions and engineering approaches.
- Accept that designs may evolve as new operational evidence becomes available.

---

## Unacceptable Behavior

The following behaviors are not acceptable:

- Personal attacks
- Harassment
- Discrimination
- Trolling
- Spam
- Deliberate misinformation
- Offensive or abusive language
- Publishing confidential information without permission
- Disrupting productive technical discussion

---

## Engineering Discussions

Engineering decisions should be based on:

- Measured results
- Operational evidence
- Technical reasoning
- Reproducible observations

Healthy technical disagreement is encouraged.

Personal conflict is not.

---

## Contributions

When contributing:

- Read the documentation in the `CODEX/` directory.
- Follow the documented engineering principles.
- Keep pull requests focused.
- Update documentation when architectural decisions change.
- Prefer clarity over cleverness.

---

## Reporting Issues

If you encounter behavior that violates this Code of Conduct, please contact the project maintainer.

Reports will be handled respectfully and, where appropriate, confidentially.

---

## Scope

This Code of Conduct applies to:

- GitHub Issues
- Pull Requests
- Discussions
- Project documentation
- Any official communication related to the EDNA OS repository

---

## Our Commitment

EDNA OS is built on the principles of engineering, transparency and continuous improvement.

We aim to create an environment where technical ideas are evaluated on their merit, collaboration is encouraged and contributors feel welcome to participate.

Thank you for helping improve EDNA OS.

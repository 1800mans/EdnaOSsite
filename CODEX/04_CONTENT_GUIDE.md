# EDNA OS Website
# Content Guide
## Version 1.0

---

# Purpose

This document defines the content strategy for every page of the EDNA OS website.

It specifies:

- page purpose
- visitor intent
- engineering message
- content hierarchy
- required sections
- calls to action
- internal links
- emotional photography

This document intentionally contains **no implementation details**.

It defines **what** should be communicated, not **how** it is implemented.

---

# Editorial Philosophy

The website should read like an engineering publication.

Not marketing.

Not sales.

Not documentation alone.

Every page should explain one engineering idea clearly.

Visitors should progressively understand:

Problem

↓

Engineering Objectives

↓

Operating Principles

↓

Measurement

↓

Architecture

↓

Validation

---

# Writing Style

Write as an engineer.

Use evidence.

Avoid exaggeration.

Never claim perfection.

Never claim superiority without explanation.

Prefer:

measured

validated

observed

implemented

evaluated

engineered

determined

Avoid:

best

world-class

revolutionary

industry-leading

AI-powered

smart

cutting-edge

next generation

# Content Depth

create another page.

# Calls to Action

Every page ends with one logical next step.

Never more than two CTAs.

Examples

Learn About Lysimeters

Read Documentation

View Deployment

Read Engineering Journal

---

# Internal Linking

Every page should link to:

Parent

Related topic

Next logical topic

Previous topic (where appropriate)

This creates both a natural user journey and strong internal SEO.

---

# Photography Rules

Every page begins with one dominant photograph.

Images must show engineering reality.

No stock technology.

No AI-generated artwork.

Real infrastructure.

Real orchards.

Real equipment.

# Final Statement

The EDNA OS website should educate visitors through a structured engineering narrative.

Each page explains one concept.

Each concept leads naturally to the next.

The result is a website that documents—not markets—the creation of a new engineering discipline:

**Orchard Operating System.**

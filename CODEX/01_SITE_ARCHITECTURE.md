# Purpose

This document defines the complete information architecture of the EDNA OS website.

It specifies:

- navigation
- page hierarchy
- user journey
- content relationships
- page responsibilities

This document does **not** define visual design or implementation details.

Those are covered elsewhere.

---

# Website Philosophy

The website should educate visitors gradually.

No page should attempt to explain everything.

Knowledge is layered.

Every page answers one primary question.

Every page naturally leads to the next question.

This principle is referred to throughout the project as:

> **The Matryoshka Principle**

Maximum navigation depth:

**3 Levels**

• Header Navigation

• Breadcrumbs

• Previous Page

• Next Page

• Related Pages

• Footer Navigation

---

# Content Rules

Each page should answer only one primary question.

Avoid duplicating explanations across pages.

Instead:

Introduce a concept.

Provide enough context.

Link to a dedicated page for deeper explanation.

# Architecture Summary

The EDNA OS website is designed as a layered engineering knowledge base.

Visitors move from understanding **why** EDNA exists, to **how** it operates, to **how** it is engineered, and finally to **how** it is validated in real commercial orchard conditions.

Every page serves a single purpose.

Every page advances the engineering narrative.

Every page reinforces the central idea:

> **EDNA OS is defining the Orchard Operating System.**
